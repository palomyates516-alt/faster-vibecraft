# Vibecraft 性能优化指南

本指南帮助您根据设备性能调整 Vibecast 设置，以获得最佳体验。

## 🎯 快速建议

### 低端设备（CPU: 2核，内存: 4GB）

**推荐配置**:
- ✅ 渲染模式: **3D** (Three.js)
- ✅ 世界网格: **关闭**
- ✅ 终端轮询: **5秒**
- ✅ 环境粒子: **关闭**
- ✅ 区域粒子: **关闭**

**预期效果**: CPU 使用率 < 15%

---

### 高端设备（CPU: 8核+，内存: 16GB+）

**推荐配置**:
- ✅ 渲染模式: **3D** (Three.js)
- ✅ 世界网格: **开启**（视觉增强）
- ✅ 终端轮询: **2秒**（更及时的反馈）
- ✅ 环境粒子: **开启**（氛围效果）
- ✅ 区域粒子: **开启**（活动指示）

**预期效果**: 完整视觉体验，CPU 使用率 20-30%

---

### 极低性能设备（CPU: 1-2核，内存: 2GB）

**推荐配置**:
- ⚠️ 渲染模式: **2D** (DOM) - 实验性功能
- ✅ 世界网格: **关闭**
- ✅ 终端轮询: **5秒**
- ✅ 环境粒子: **关闭**
- ✅ 区域粒子: **关闭**

**预期效果**: CPU 使用率 < 5%

> **注意**: 2D 模式为实验性功能，可能缺少部分 3D 特性

## ⚙️ 访问性能设置

### 方法 1: 通过设置菜单

1. 点击右上角的 **齿轮图标** (Settings)
2. 找到 **"Performance"** 部分
3. 调整各项设置
4. 关闭模态框自动保存

### 方法 2: 通过浏览器控制台

按 `F12` 打开控制台，然后：

```javascript
// 切换渲染模式（需要重新加载页面）
localStorage.setItem('vibecraft-performance-config', JSON.stringify({
  renderMode: '2d',
  worldGrid: false,
  terminalPollInterval: 5000,
  enableAmbientParticles: false,
  enableZoneParticles: false
}))
location.reload()
```

## 📋 设置选项说明

### 渲染模式 (Render Mode)

**3D (Three.js)** - 默认推荐
- 完整的 3D 视觉体验
- 六边形区域、站点模型、角色动画
- CPU 使用率: 10-20%

**2D (DOM)** - 实验性
- 纯 CSS/DOM 渲染，无 WebGL
- 使用 emoji 图标代替 3D 模型
- CPU 使用率: < 5%
- ⚠️ 某些功能可能受限

### 世界网格 (World Grid)

- **开启**: 显示世界六边形网格覆盖整个地板
  - 视觉增强，帮助理解空间布局
  - CPU 影响: +10-15%

- **关闭**: 不显示网格（默认）
  - 更清洁的外观
  - 节省 CPU 和内存

### 终端轮询 (Terminal Polling)

控制终端输出面板更新频率：

- **2 秒**: 更及时的反馈，CPU 使用率 +2-5%
- **5 秒**: 降低 CPU 使用率（推荐默认）
- **10 秒**: 最低 CPU 使用率，但反馈延迟

### 环境粒子 (Ambient Particles)

场景中的漂浮粒子效果：

- **开启**: 增加氛围感，CPU 使用率 +5-10%
- **关闭**: 禁用粒子（默认）

### 区域粒子 (Zone Particles)

Zone 活动时的粒子效果：

- **开启**: 视觉活动指示，CPU 使用率 +5-10%
- **关闭**: 禁用粒子

## 🔧 故障排除

### 问题: 设置未保存

**原因**: 浏览器隐私模式或 localStorage 被禁用

**解决**:
1. 检查浏览器是否在隐身/隐私模式
2. 允许浏览器使用 localStorage
3. 刷新页面重试

### 问题: 切换到 2D 模式后仍显示 3D

**原因**: 2D 渲染器为实验性功能

**当前行为**:
- 2D 模式会保存到 localStorage
- 但实际渲染仍使用 3D（暂未完全集成）
- 页面会显示提示 "2D mode is experimental. Using 3D renderer."

**未来版本**: 将完全支持 2D 模式切换
